📌 初期構成ファイル・ディレクトリパス

RHEL/CentOS環境における主要なファイル配置は以下の通りです。次のステップとして構成ファイルの編集を開始できます。

📁 /etc/nginx/nginx.conf

メインの構成ファイル。グローバル設定、イベント駆動、HTTPコンテキスト全体を司ります。

📁 /usr/share/nginx/html

デフォルトのドキュメントルート。現在表示されているこのHTMLはこの配下に格納されています。

📁 /var/log/nginx/access.log

クライアントからのHTTPリクエストがリアルタイムに記録されるデフォルトアクセスログ。

📁 /var/log/nginx/error.log

nginxサーバー自体の動作エラーやリクエスト失敗の原因が記録されるエラーログ。

⌨️ サービス管理用の基本コマンド

サーバー起動、状態確認、および構成変更の反映に用いるシステムコマンドです。

Web Server Console 
# nginx サービスの開始
[root@webserver ~]# systemctl start nginx

# 起動状態とプロセスの稼働ステータス確認
[root@webserver ~]# systemctl status nginx

# OS起動時に自動的にnginxサービスを立ち上げる有効化設定
[root@webserver ~]# systemctl enable nginx

# 設定ファイル変更後の整合性（シンタックスエラー）チェック
[root@webserver ~]# nginx -t
nginx: the configuration file /etc/nginx/nginx.conf syntax is ok
nginx: configuration file /etc/nginx/nginx.conf test is successful

# 接続を切断せずに設定を即座に適用するリロード
[root@webserver ~]# systemctl reload nginx

🌐 バーチャルホスト (Server Blocks) の設定

同一の物理サーバー上で、ドメイン名ごとに異なるディレクトリのコンテンツを配信（仮想ホスト）する最も一般的な構成例です。

推奨設定方法として、/etc/nginx/conf.d/ ディレクトリに .conf 拡張子で設定を追加します。メインの nginx.conf でこれらのファイルが自動でインクルードされるように定義されています。

/etc/nginx/conf.d/example.conf 
# 新しいドメイン向けのサーバーブロック構成例
server {
    listen       80;
    server_name  example.com www.example.com;

    # ドキュメントルートディレクトリの指定
    root         /var/www/example.com/html;
    index        index.html index.htm;

    # URLのルーティング設定とリクエストのマッピング
    location / {
        try_files $uri $uri/ =404;
    }

    # 404エラーページのカスタム設定
    error_page 404 /404.html;
    location = /404.html {
        root   /usr/share/nginx/html;
    }
}

🔒 リバースプロキシとSSL/TLS（HTTPS）の構成

Node.jsやPython(Gunicorn)、Goなどのアプリケーションサーバーの手前に配置し、暗号化通信の終端とロードバランシングを行うリバースプロキシ設定のテンプレートです。

/etc/nginx/conf.d/api_proxy.conf 
# 1. HTTPアクセスを自動的にHTTPS（常時SSL）へ301リダイレクトする
server {
    listen       80;
    server_name  api.example.com;
    return 301   https://$host$request_uri;
}

# 2. HTTPS通信を終端し、ローカルのポート3000へプロキシ転送する
server {
    listen       443 ssl http2;
    server_name  api.example.com;

    # SSL証明書のパス設定
    ssl_certificate     /etc/pki/tls/certs/api_example.crt;
    ssl_certificate_key /etc/pki/tls/private/api_example.key;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_ciphers         HIGH:!aNULL:!MD5;

    # プロキシフォワード設定
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

⚡ セキュリティとパフォーマンス最適化

コンテンツ転送速度の向上と、セキュアなヘッダー設定のための推奨オプションです。

/etc/nginx/conf.d/security.conf 
# 1. バージョン情報を非表示にし攻撃対象としての露出を低減
server_tokens off;

# 2. XSSやクリックジャッキング防御用セキュリティヘッダーの設定
add_header X-Frame-Options "SAMEORIGIN" always;
add_header X-Content-Type-Options "nosniff" always;
add_header X-XSS-Protection "1; mode=block" always;

# 3. Gzip圧縮によるアセットファイル転送サイズ軽量化
gzip            on;
gzip_types      text/plain text/css application/json application/javascript text/xml;
gzip_min_length 1000;

# 4. アップロード可能サイズの上限緩和設定 (デフォルトは1M)
client_max_body_size 20M;
