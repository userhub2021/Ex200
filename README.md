RHCSA (EX200) RHEL 10 対応試験 概要と対策ガイド本ドキュメントは、Red Hat Enterprise Linux 10 (RHEL 10) のリリースに伴ってアップデートされた Red Hat Certified System Administrator (RHCSA) 試験 (EX200 v10) の概要、主な変更点、および出題範囲について解説します。1. 試験の基本情報RHCSA（EX200）は、Linuxシステム管理における最も定評のある実技形式（パフォーマンスベース）の試験です。試験時間: 3時間問題形式: 実技試験（実機エミュレータを使用し、提示される要件に従ってシステムを構成・修正する）合格基準: 300点満点中 210点以上（約70%の得点率）前提知識: 基本的なコマンドライン操作、ファイルシステム、ネットワーク、ユーザー管理などの基礎知識合格後の有効期限: 3年間2. RHEL 9 から RHEL 10（v10）への主な変更点RHEL 10 試験では、現代のLinuxシステム管理の実態に合わせて、試験範囲の大幅な見直しが行われました。RHEL 9向けの教材を使用している場合は特に注意が必要です。🟢 追加された主要なトピックFlatpak パッケージの管理デスクトップアプリなどで標準となりつつある「Flatpak」が追加されました。Flatpakリポジトリ（Flathubなど）の追加/削除、アプリケーションの検索・インストール・削除、システムレベルとユーザーレベルでのインストールの違いを理解する必要があります。systemd タイマー (systemd Timers) の作成従来の cron や at 試験対策のアドバイス（重要チェックポイント）すべての構成は「再起動後も有効」でなければならない実技試験の採点システムは、受験者がすべての作業を終えた後、システムを強制再起動してからテストを実行します。マウント（/etc/fstab）、ネットワーク設定、systemdサービス/タイマー、SELinux設定などは、再起動後に正しくロードされないと「0点」になります。設定後は必ず自分で再起動テストを行ってください。ヘルプとドキュメント（man, info）の活用試験は完全にオフラインで、インターネット検索はできません。しかし、ローカルの man ページや /usr/share/doc/ 内のサンプルコード（特に autofs や systemd.timer の構成例）は自由に閲覧できます。コマンドのオプションやファイルの書き方に迷ったら、すぐに man を引けるように準備しておきましょう。Flatpak と systemd タイマーの練習を重ねるRHEL 10で新しく追加されたこの2つのテーマは、出題される可能性が非常に高いです。特に systemd.timer の OnCalendar= 構文（例：毎週月曜日の午前9時に実行など）の指定方法をしっかりと練習しておきましょう。
